食谱规划（meal-planner-skill）· 人类读本

从你的菜品库排一周食谱，并把结果写入按月归档的饮食计划文件。本页只描述本 skill 的分工与路径；完整步骤与约束见同目录 SKILL.md。

简介

meal-planner-skill 让 Agent 读取你在 skill 目录下的用户档案与菜品详情库，只在你明确点名本 skill 时，按「当周期望」从 dish-library.md 选菜并写入 meal-plan-YYYY-MM.md。

随口聊吃喝、不问正式排餐时，不应自动跑本 skill——详见下文「何时点名调用」。

详细介绍

何时点名调用

仅在对话里明确提到 meal-planner-skill（或等价语句）时启用排餐工作流。泛泛问菜谱、热量、做法而不点名本 skill 时，按普通问答处理。

  • 示例触发：「用 meal-planner-skill 帮我排下周」「调用食谱规划 skill」。
  • 不确定时先问一句：是要正式读写档案与菜品库排一周，还是随便聊聊。

数据放在哪

  • 永远在 skill 目录：user-profile.md、dish-details.md、templates/、本 skill 说明文件。
  • 仅在「数据根目录」DATA_ROOT（由 user-profile.md 第 0 节配置）：dish-library.md（菜名）与 meal-plan-YYYY-MM.md（按月计划）。除此二类外勿在用户目录新建其它文件。
  • 跨月的一周归属：看该周周一落在哪个月，详见 SKILL.md Step 5。

工作流概要

Agent 按 SKILL.md 中的 Step 0→6：解析数据根 → 是否首次建档 → 解析当周期望 → 检查菜品库 → 写入月份文件 → 聊天里给摘要并询问是否迭代。缺菜时先补库再排餐。
